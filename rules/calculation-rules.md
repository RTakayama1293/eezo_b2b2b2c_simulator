# 計算ルール

## 金額・単位の取り扱い

### 内部処理
- すべての金額は「円」単位で処理
- 浮動小数点演算による誤差に注意

### 表示時
- 100万円以上: 万円表示（小数点1桁）
- 100万円未満: 円表示（カンマ区切り）

### 変換関数
```python
def to_man_yen(yen: float) -> float:
    return yen / 10000
```

## LTV計算

### BtoB顧客のLTV

```
LTV = 年間取引額 × Σ(継続率^n) for n=0 to 継続年数-1
```

**前提**:
- 継続率は毎年一定と仮定
- 取引額は初年度と同額と仮定

**例**: 年間120万円、継続率80%、3年継続
```
LTV = 120万 × (1 + 0.8 + 0.64) = 120万 × 2.44 = 292.8万円
```

### EC顧客のLTV

```
LTV = 顧客単価 × 年間購入頻度 × 継続年数 × リピート補正
```

**前提**:
- リピート率は初回購入者に対する3ヶ月以内の割合
- 継続年数は平均値（中央値ではない）

## CAC計算

### BtoB営業

```
CAC = (固定費 + 訪問変動費 + 成約変動費) / 新規成約数
```

**含まれるコスト**:
- 人件費（社会保険込み）
- 交通費
- サンプル費
- 通信費
- CRM利用料
- 契約事務費

### EEZO（EC）

```
CAC = (広告費 + インセンティブ) / 新規顧客数
```

**含まれるコスト**:
- デジタル広告費
- 店舗インセンティブ
- プラットフォーム費は含まない（固定費として別計上）

## シナジー効果の適用ルール

### 適用範囲

| 効果 | 適用先 | 乗数範囲 |
|------|--------|----------|
| CVR向上（BtoB） | 成約率 | 1.0〜1.5 |
| CVR向上（EEZO） | QR→購入CVR | 1.0〜1.5 |
| LTV向上 | 加重平均LTV | 1.0〜1.5 |
| コスト効率化 | 固定費 | 0.7〜1.0 |

### 計算順序
1. 基準シナリオの値を計算
2. シナジー乗数を適用
3. 追加コストを加算

**注意**: シナジー効果は楽観的になりがち。初期分析では1.1〜1.2程度を推奨。

## 損益分岐点（BEP）計算

### 基本式

```
BEP（月数）= 初期投資額 / 月間利益
```

### 初期投資の範囲

**BtoB単独**: なし（オペレーション費用のみ）

**EEZO単独**: 
- プラットフォーム初期設定費

**同時並行**:
- 店舗セットアップ費 × 店舗数
- 連携システム構築費

### 判定基準
- 12ヶ月以内: 良好
- 12〜24ヶ月: 要検討
- 24ヶ月超: 要再検討

## ROI計算

### 3年ROI

```
ROI = (3年間売上 - 3年間コスト - 初期投資) / (3年間コスト + 初期投資)
```

### 割引率の適用（任意）
- 割引率を適用する場合はNPVベースで計算
- 推奨割引率: 10%/年（月次: 0.8%）

## 感度分析のルール

### 変動範囲
- 標準: ±30%（基準値の0.7〜1.3倍）
- 不確実性が高い場合: ±50%

### 分析対象パラメータ（優先順）
1. CVR（最重要）
2. 顧客単価
3. リピート率
4. シナジー乗数
5. インセンティブ率

### 結果の解釈
- **頑健**: 最悪ケースでもLTV/CAC > 3
- **妥当**: 基準ケースでLTV/CAC > 4、最悪ケースでも > 2
- **リスキー**: 基準ケースでもLTV/CAC < 4

## 検証チェックリスト

### 計算結果の妥当性確認
- [ ] LTV/CACが10を超えていないか（過大評価の可能性）
- [ ] 月間利益が売上の50%を超えていないか（コスト漏れの可能性）
- [ ] 損益分岐点が1ヶ月未満になっていないか（計算ミスの可能性）
- [ ] シナジー効果適用後の値が現実的か
